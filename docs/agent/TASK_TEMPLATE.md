# 智能体任务模板

复制本模板到 issue、PR 描述或任务文档中，用来交给智能体执行。

```md
# 任务标题

## 目标

一句话说明要达成的结果。

## 背景

为什么要做这件事，相关用户场景或故障现象是什么。

## 范围

需要修改的模块、接口或文档：

- 

明确不在本次处理的内容：

- 

## 相关记忆

开始前至少读取：

- `AGENTS.md`
- `docs/agent/MEMORY.md`
- `SYSTEM_DESIGN_SPEC.md` 的相关章节
- 相关源码文件：

## 约束

例如兼容性、性能、字符设备协议、内核版本、硬件 offload 或部署限制。

## 验证标准

必须执行或说明无法执行的验证：

- `git diff --check`
- `make`
- 其他：

## 交付物

期望输出：

- 代码改动
- 文档更新
- 决策记录
- 验证结果

## 退出条件

满足哪些条件就认为任务完成。

## 交接备注

留给下一轮智能体或维护者的注意事项。
```
